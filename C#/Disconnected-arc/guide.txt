C# Windows Forms – Disconnected Architecture (ADO.NET)
UI Design Guide (Toolbox Elements)

---

1. LABELS (3 Required)

---

Drag: Label

Set properties:

* Label1 → Text = "Id"
* Label2 → Text = "Name"
* Label3 → Text = "City"

---

2. TEXTBOXES (3 Required)

---

Drag: TextBox

Set (Name property):

* textId       (⚠ Used for Update/Delete only)
* textName
* textCity

---

3. BUTTONS (4 Required)

---

Drag: Button

Set properties:

1. Name = button1
   Text = Insert

2. Name = button2
   Text = Update

3. Name = button3
   Text = Delete

4. Name = button4
   Text = Refresh / Select

---

4. DATAGRIDVIEW (1 Required)

---

Drag: DataGridView

Set:

* Name = dataGridView1

---

5. EVENT CONNECTION (IMPORTANT)

---

Double click each button:

* button1 → button1_Click (Insert into DataSet)
* button2 → button2_Click (Update DataSet)
* button3 → button3_Click (Delete from DataSet)
* button4 → button4_Click (Refresh Data)

---

6. SPECIAL NOTE (DISCONNECTED LOGIC)

---

✔ Data is NOT directly stored in database
✔ Data is first stored in DataSet (memory)
✔ SqlDataAdapter syncs DataSet ↔ Database

✔ Id is AUTO-GENERATED by database
→ Do NOT enter Id while inserting

✔ textId is used ONLY for:

* Update
* Delete

---

## FINAL UI STRUCTURE

[Label] Id      → [TextBox: textId]
[Label] Name    → [TextBox: textName]
[Label] City    → [TextBox: textCity]

[Insert] [Update] [Delete] [Refresh]

[DataGridView → dataGridView1]

---

## EXAM MEMORY TRICK

Connected → Direct DB (SqlCommand)

Disconnected → DataSet + DataAdapter

Flow:
UI → DataSet → DataAdapter → Database

---

## IMPORTANT CONTROL NAMES

textId, textName, textCity
button1, button2, button3, button4
dataGridView1
